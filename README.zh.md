给任何一个真实的人——前任、老板、暧昧对象、难搞的客户、爸妈——建一个**赛博影子**。

然后你可以做三件事:

🕯️ **审讯模式 `/mindreader seance`** — 翻出过去的某段对话,问影子:*"TA 当时到底在想什么?"*

🎭 **预演模式 `/mindreader rehearse`** — 在和真人开口前,先把你要说的话在影子身上跑一遍。

📐 **校准模式 `/mindreader calibrate`** — 喂一段真实对话,影子猜后半段,看它猜得有多准。

> 这是一个**假设生成器**,不是真的读心术。它不会告诉你"真相",它会告诉你**你没想到的可能性**。区别很重要,下面有一整段在讲为什么。

## 🎬 Demo

### 🕯️ `/mindreader seance` — *"TA 那句话到底什么意思?"*

你翻出三个月前的一段对话,死活想不通 TA 那句"随便吧"到底什么意思。把对话片段喂进去:

```
你           ❯ /mindreader seance

mindreader   ❯ 把那段对话粘进来,再告诉我你卡在哪里。

你           ❯ [粘贴 20 条聊天记录]
             ❯ 我不懂 TA 最后那句"随便吧"是赌气还是真的不在乎了

mindreader   ❯ 我从这 20 条里读到 3 种相互矛盾的可能性。
             ❯ 我不知道哪个是真的。你比我更可能知道。
             ❯
             ❯ ━━━ 解读 A:在赌气 (置信度 ~40%)
             ❯ 证据:第 7 条 TA 主动提了周末安排,你回得很短;
             ❯       第 12 条 TA 用了句号结尾,这在你们之前的
             ❯       聊天里只在 TA 不爽时出现过 3 次。
             ❯ 如果是这个解读,TA 在等你追问。
             ❯
             ❯ ━━━ 解读 B:真的累了 (置信度 ~35%)
             ❯ 证据:第 3 条 TA 提了加班,第 15 条之后回复间隔
             ❯       从平均 2 分钟拉到 18 分钟。
             ❯ 如果是这个解读,追问会让 TA 更烦。
             ❯
             ❯ ━━━ 解读 C:其实没想那么多 (置信度 ~25%)
             ❯ 证据:你们之前有 4 次类似句式都没有后续冲突。
             ❯       你可能在用现在的焦虑反推当时的语气。
             ❯
             ❯ ⚠️  这三个我都能编得很像真的。这是 LLM 的强项,
             ❯     也是它的陷阱。请把这当成"我没想到的可能性"
             ❯     列表,不是答案。
```

**关键设计:影子不会给你一个答案,它会给你三个矛盾的答案 + 证据 + 置信度 + 自我警告。** 这是这个工具和算命软件的分水岭。下面有一整段讲为什么。

### 🎭 `/mindreader rehearse` — *"在真的开口之前,先在影子身上跑一遍。"*

你想跟老板提加薪,但你也知道老板是那种一被冒犯就翻脸的人。先在影子身上跑一遍:

```
你           ❯ /mindreader rehearse

mindreader   ❯ 目标是什么?(一句话)

你           ❯ 让老板同意给我涨 20%,不要把关系搞僵

mindreader   ❯ 你打算怎么开口?

你           ❯ "老板,我觉得我这一年贡献挺大的,想聊聊薪资"

mindreader   ❯ 跑了 5 轮模拟,目标达成度 2/10。
             ❯
             ❯ ━━━ 影子的反应
             ❯ 影子: "贡献大?具体哪些?"
             ❯ 影子内心 OS: (又来一个觉得自己很重要的)
             ❯
             ❯ ━━━ 失败模式
             ❯ 你触发了 TA 的"被索取"防御。在 TA 的认知里,
             ❯ "贡献大"是结论,不是论据。TA 要的是论据。
             ❯
             ❯ ━━━ 改写建议 (达成度预估 7/10)
             ❯ "老板,我想跟你对一下今年我那几个项目的产出,
             ❯  顺便聊聊接下来一年的方向和定位。"
             ❯
             ❯ 这个版本绕开了"涨薪"这个触发词,把对话框架从
             ❯ "我要"换成"我们对齐",更符合 TA 的沟通偏好。
             ❯ 涨薪的话题在第二轮再提,等 TA 已经认可了框架。
             ❯
             ❯ ⚠️  影子是基于你给的聊天记录建的。如果记录少,
             ❯     或者只覆盖一种场景,模拟会失真。这是预演,
             ❯     不是脚本。真人面前请保持临场判断。
```

**关键设计:每一轮都标注影子的"内心 OS",每一次输出都附带失真警告。** 这两个细节是娱乐性和严肃性的分水岭。

### 📐 `/mindreader calibrate` — *"我的影子到底有多准?"*

你给前任建了个影子,但它到底准不准?拿一段真实对话来测:

```
你           ❯ /mindreader calibrate

mindreader   ❯ 粘贴一段完整的真实对话(至少 10 条消息)。

你           ❯ [粘贴 30 条聊天记录]

mindreader   ❯ 收到 30 条消息。
             ❯
             ❯ 拆分方案:
             ❯   前半段:第 1~15 条(喂给影子)
             ❯   后半段:第 16~30 条(影子来猜)
             ❯
             ❯ 开始校准?

你           ❯ 开始

mindreader   ❯ ━━━ 影子校准报告 ━━━
             ❯
             ❯ 📊 总分:7.2/10
             ❯
             ❯ 30 条消息(前半段 15 条 / 后半段 15 条)
             ❯ 影子预测了 8 条 TA 的回复
             ❯
             ❯ ━━━ 逐条对比 ━━━
             ❯
             ❯ 第 17 条(TA):
             ❯   真实:"哈哈行行行随便你"
             ❯   影子:"哈哈好好好你说了算"
             ❯   评分:8/10
             ❯   分析:语气一致,都是敷衍式撒娇,
             ❯         影子抓住了情绪但用词略有偏差。
             ❯
             ❯ 第 22 条(TA):
             ❯   真实:"现在别聊这个行吗"
             ❯   影子:"我不太想讨论这个"
             ❯   评分:7/10
             ❯   分析:同样是回避,但影子太书面了,
             ❯         真实版本更口语更急。
             ❯
             ❯ ━━━ 维度评分 ━━━
             ❯
             ❯   语义方向:7.8/10  ████████░░
             ❯   情绪基调:7.5/10  ████████░░
             ❯   表达风格:6.2/10  ██████░░░░
             ❯   行为选择:7.0/10  ███████░░░
             ❯
             ❯ ━━━ 诊断 ━━━
             ❯
             ❯ 强项:影子对 TA 的情绪方向判断准确,
             ❯       知道 TA 什么时候会回避、什么时候会接话。
             ❯ 盲区:表达风格偏书面,真人更口语化、更多缩写。
             ❯ 建议:追加更多日常闲聊记录,让影子学 TA 的语感。
             ❯
             ❯ ⚠️  这个分数不代表"你有多了解 TA"。
             ❯     它只代表:基于你喂的数据,影子能在多大程度上
             ❯     复现 TA 的表达。数据偏了,分数就偏。
             ❯
             ❯ 📸 截图发朋友圈?
             ❯    "我的影子猜中了 72%,你的呢?"
             ❯    #mindreader #数字影子
```

**关键设计:分数拆成四个维度 + 逐条对比,让你看到影子_哪里准哪里偏_。** 当然,照例附带警告——分数衡量的是影子的数据质量,不是你对 TA 的了解程度。

## 🧠 为什么做这个(以及为什么这么做)

直说:这个项目踩在伦理的灰色地带,我们知道。

**关于审讯模式:**

LLM 不可能"知道"另一个人当时在想什么。它只见过你提供的语料和你的描述。所以如果它给你一个**单一的"真相"**,那个真相 100% 是幻觉——一个用 TA 的语气重新包装过的、你自己**已经隐隐怀疑但没明说**的解读。

这就是为什么 `/seance` 强制输出**多个矛盾解读 + 证据 + 置信度 + 自我警告**。它的设计目的是**松动**你固化的叙事,而不是**加固**它。如果你只想要一个肯定的答案,这个工具会让你失望——这是故意的。

**关于预演模式:**

"在影子身上奏效的话术"和"在真人身上奏效的话术"之间有差距,这个差距等于影子模型相对于真人的所有偏差之和。你优化得越狠,越可能收敛到**利用模型缺陷**而不是**真正击中那个人**的句子。这叫 Goodhart's law,在这个领域特别致命。

所以 `/rehearse` 不让你跑 1000 轮选最优,它最多跑 5 轮,然后强制提醒你**这是预演,不是脚本**。预演的目的是让你**在真实对话里更不容易失控**,而不是预先写好台词。

**元批评:**

是的,把人当成可以反向工程的系统,本身就是一种关系姿态。哪怕对方不知道,你知道。这个工具会改变你怎么看待你身边的人。这是真实的代价,我们不假装它不存在。

如果你看完这三段还想用,欢迎。如果你看完觉得"这玩意儿就该被烧掉",也欢迎——issue 区敞开,我们认真回复。

## 🚀 快速开始

```bash
# 安装 (Claude Code,全局)
git clone https://github.com/chkfail/mindreader-skill ~/.claude/skills/mindreader

# 或安装到当前项目
mkdir -p .claude/skills && \
  git clone https://github.com/chkfail/mindreader-skill .claude/skills/mindreader
```

**第一次使用:**

```
/mindreader build         # 建一个新的影子
/mindreader seance        # 审讯模式
/mindreader rehearse      # 预演模式
/mindreader calibrate     # 校准影子准确度
/mindreader list          # 看你建过哪些影子
/mindreader forget <n>    # 删除一个影子
```

**喂数据:**

- 直接粘贴聊天记录文本(任意格式,影子会自己解析)
- 或导入 [ex-skill](https://github.com/titanwings/ex-skill) 已经生成的 persona 文件
- 或纯靠描述,什么记录都不给也能跑(质量会下降)

## 🧬 工作原理

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   你的聊天记录 + 你的描述                               │
│            │                                            │
│            ▼                                            │
│   ┌─────────────────┐                                   │
│   │  人格构建器     │  (复用 ex-skill 的六层结构)        │
│   └─────────────────┘                                   │
│            │                                            │
│            ▼                                            │
│   ┌─────────────────┐                                   │
│   │   Shadow 影子   │                                   │
│   └─────────────────┘                                   │
│       │           │          │                           │
│       ▼           ▼          ▼                           │
│  ┌─────────┐ ┌──────────┐ ┌────────────┐                │
│  │ /seance │ │/rehearse │ │/calibrate  │                │
│  └─────────┘ └──────────┘ └────────────┘                │
│  多解读+证据  5轮模拟+OS   拆分预测+评分                │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

影子的人格结构基于 [ex-skill](https://github.com/titanwings/ex-skill) 的六层模型 (核心模式 → 身份 → 表达风格 → 情感行为 → 冲突边界 → 雷区),感谢原作者。

## ❓ FAQ

**Q: 这玩意儿合法吗?**
A: 工具本身合法。你怎么用它是你的事。请不要喂任何不属于你自己的聊天记录,请不要拿它去骚扰、操纵或欺骗任何人。法律和道德的边界你自己守。

**Q: 准确度有多高?**
A: 不知道,也没法衡量,因为没有 ground truth。这是一个假设生成工具,不是预测工具。如果你想要"科学准确度",这个项目不适合你。

**Q: 和 ex-skill 是什么关系?**
A: ex-skill 是这个项目的精神祖先,人格生成部分直接复用了它的方法论。区别是 ex-skill 的对象是已经离开你生活的人(哀悼),mindreader 的对象是还在你生活里的人(认知)。如果你只想和前任的影子聊天,直接用 ex-skill,它做得更好。

**Q: 我能用它对付我老板吗?**
A: 技术上可以。心理上你要想清楚:当你开始把身边所有人都看成可以反向工程的系统,你也在变成那个系统。这不是劝退,是提醒。

**Q: 影子会不会被滥用来 PUA 别人?**
A: 会。任何理解人的工具都可以被用来操纵人——从《孙子兵法》到《人性的弱点》到所有心理学教材。我们的对策是:(1) 强制多解读输出,降低"我抓到 TA 的把柄了"的幻觉;(2) 预演模式限制最多 5 轮,降低过拟合;(3) README 这一整段。剩下的靠你自己。

**Q: 数据存在哪?会不会上传?**
A: 全本地。所有影子文件存在 `~/.claude/skills/mindreader/shadows/` 下。我们没有服务器,没有 telemetry,没有云端。你删了就真没了。

**Q: 支持其他语言吗?**
A: Prompts 是中英双语的,影子可以用任何语言建。文档目前中英双语,其他语言欢迎 PR。

**Q: 影子会不会以一种伤害我的方式给出错误判断?**
A: 会,而且很容易。特别是当聊天记录只覆盖一种场景的时候(比如全是吵架记录,或者全是甜蜜期记录),影子会过拟合到那个切片然后自信地外推。每次输出附带的失真警告不是 boilerplate,是承重墙,请认真读。
