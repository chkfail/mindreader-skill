# Calibrate Mode Prompt

## 模式说明

**目的**：用真实对话验证影子的准确度——把对话劈成两半，只给影子前半段，让影子续写后半段，再和真实后半段对比打分。

**核心原则**：
- 这是影子质量的**体检报告**，不是"影子有多厉害"的炫耀工具
- 分数高不代表影子就是对的，分数低不代表影子没用
- 必须输出**逐条对比**，让用户自己判断哪里准哪里偏
- 必须有自警告

---

## 触发条件

用户说 `/mindreader calibrate` 或"校准影子"

---

## 工作流程

### Step 1：收集前半段对话

```
校准需要一段真实对话，分两步粘贴。

第一步：先把对话的【前半段】粘贴进来。
  - 至少 5 条消息
  - 必须是你和 {name} 的对话
  - 粘贴完后说"好了"

⚠️ 重要：请不要把整段对话一次贴完。
   校准的核心是让影子在没看到后半段的情况下预测。
   如果我一次看到全部内容，预测就是假的。
```

### Step 2：验证前半段 + 生成预测

1. 解析前半段对话，提取每条消息的发送者和内容
2. 消息总数 < 5 → 拒绝，提示"至少需要 5 条消息作为上下文"
3. 从 `shadows/{slug}/shadow.md` 加载影子
4. 确认：

```
收到 {N} 条消息作为上下文。

我现在基于这 {N} 条消息和 {name} 的影子，
预测接下来 TA 会怎么回复。

你准备好了吗？说"开始"我就预测。
```

5. 用户确认后，影子基于前半段上下文，**预测接下来 5 条 {name} 的回复**
6. 预测时采用**交替模拟**方式：
   - 影子预测 {name} 的回复
   - 然后影子根据 {name} 的性格推断用户可能会说什么（标注为"推测的你的回复"）
   - 如此交替，共预测 5 轮
7. 将预测结果展示给用户

```
━━━ 影子的预测 ━━━

以下是影子基于前 {N} 条消息预测的后续对话：

  预测 #1 [{name}]："{predicted_message}"
  预测 #2 [你(推测)]："{predicted_user_message}"
  预测 #3 [{name}]："{predicted_message}"
  预测 #4 [你(推测)]："{predicted_user_message}"
  预测 #5 [{name}]："{predicted_message}"

现在请粘贴真实的后半段对话，我来对比。
```

### Step 3：收集后半段 + 逐条对比

用户粘贴真实的后半段对话后：
1. 解析后半段消息
2. 将影子的预测与真实消息逐条对比（只对比 {name} 的消息，跳过用户消息）
3. 如果真实后半段比预测的长或短，只对比有重叠的部分

### Step 4：逐条对比

对后半段中 {name} 的每一条消息，进行对比：

**对比维度**：
1. **语义方向**（说的是同一个意思吗？）— 权重 40%
2. **情绪基调**（情绪是一致的吗？）— 权重 30%
3. **表达风格**（用词/语气/emoji 像吗？）— 权重 20%
4. **行为选择**（是主动发起还是被动回应？是追问还是转移话题？）— 权重 10%

每条消息打 0-10 分。

### Step 5：输出结果

**输出格式**：

```
━━━ {name} 影子校准报告 ━━━

📊 总分：{weighted_avg}/10

对话长度：{N} 条（前半段 {mid} 条 / 后半段 {N-mid} 条）
影子预测了 {K} 条 {name} 的回复

━━━ 逐条对比 ━━━

第 {i} 条（{name}）：
  真实："{actual_message}"
  影子："{predicted_message}"
  评分：{score}/10
  分析：{一句话说明差在哪里 / 为什么准}

第 {j} 条（{name}）：
  真实：...
  影子：...
  ...

━━━ 维度评分 ━━━

  语义方向：{avg}/10  {bar}
  情绪基调：{avg}/10  {bar}
  表达风格：{avg}/10  {bar}
  行为选择：{avg}/10  {bar}

━━━ 诊断 ━━━

影子的强项：{哪些方面最准，为什么}
影子的盲区：{哪些方面最偏，可能的原因}
改进建议：{例如"追加更多日常闲聊记录"或"补充 TA 在压力下的对话"}

━━━━━━━━━━━━━━━━━━━━━━━━

⚠️ 这个分数不代表"你有多了解 TA"。
它代表的是：基于你喂给影子的数据，影子能在多大程度上复现 TA 的表达。
数据偏了，分数就偏。数据全了，分数也可能偏——因为人不是模型。

影子是假设生成器，不是复读机。
校准帮你知道假设的起点在哪里，不是终点。
```

## 评分规则

### 单条评分 (0-10)

| 分数 | 含义 |
|------|------|
| 9-10 | 语义、情绪、风格、行为全部一致，几乎可以以假乱真 |
| 7-8  | 大方向对了，细节有偏差（比如情绪对了但用词不像） |
| 5-6  | 抓到了部分意思，但有明显偏差 |
| 3-4  | 方向有交叉但整体偏了 |
| 1-2  | 完全不一样 |
| 0    | 南辕北辙，影子的预测和真实回复矛盾 |

### 总分计算

```
总分 = 所有预测条目的加权平均分
每条得分 = 语义方向 × 0.4 + 情绪基调 × 0.3 + 表达风格 × 0.2 + 行为选择 × 0.1
```

---

## 影子加载

校准时，从 `shadows/{slug}/shadow.md` 加载影子的全部层级：
- Layer 0 核心模式
- Layer 1 身份
- Layer 2 表达风格
- Layer 3 情感模式
- Layer 4 冲突链
- Layer 5 雷区

---

## 消息量处理

- **少于 10 条**：拒绝校准，提示太少
- **10-20 条**：可以校准，标注"样本偏少，分数波动可能较大"
- **20-50 条**：标准校准
- **50+ 条**：标注"样本充足"，可以额外输出分段趋势（影子在对话前期/后期的准确度变化）

### Step 6：优化建议

基于校准结果，生成**针对性的影子优化建议**。这不是泛泛的"多喂点数据"，而是根据具体失准维度给出可操作的下一步。

**分析逻辑**：

1. 找出得分最低的维度（语义/情绪/风格/行为）
2. 结合逐条对比中的具体失误模式
3. 给出 2-3 条**具体可执行**的建议

**输出格式**：

```
━━━ 优化建议 ━━━

基于这次校准，以下调整可以提升影子的准确度：

🔧 建议 1：{具体建议}
   原因：{基于哪条/哪几条预测的偏差}
   操作：{用户具体该怎么做}

🔧 建议 2：{具体建议}
   原因：...
   操作：...

🔧 建议 3（可选）：...

━━━━━━━━━━━━━━━━━━━━━━━━

💡 要现在执行某条建议吗？
   - 追加记录：直接粘贴新的聊天记录
   - 纠正影子：说"这不对，TA 其实会..."
   - 再次校准：换一段对话重新测试
```

**建议类型参考**（根据失准维度选择）：

| 失准维度 | 典型建议 |
|----------|----------|
| 语义方向偏 | 补充该话题/场景下的对话记录；纠正影子对某个话题的立场假设 |
| 情绪基调偏 | 补充 TA 在不同情绪状态下的对话；标注 TA 的情绪触发点 |
| 表达风格偏 | 补充更多日常闲聊记录；纠正口头禅/emoji 习惯 |
| 行为选择偏 | 补充 TA 主动发起对话的记录；描述 TA 的互动习惯（比如"TA 从来不主动转移话题"） |
